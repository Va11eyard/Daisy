# Training data layout

- **`raw/`** — curated JSON (or JSONL) for [scripts/prepare_dataset.py](../scripts/prepare_dataset.py). See [docs/DATASET.md](../docs/DATASET.md). Large intermediate **`raw/md_dialogues.json`** is gitignored (regenerate locally).
- **`train.jsonl` / `val.jsonl`** — generated by `prepare_dataset.py` (git-tracked when present). Regenerate after editing `raw/` or the MD corpus.

```powershell
# По умолчанию «rich»: больше и разнообразнее диалоги (см. docs/DATASET.md). Для старого режима: --quality standard
python scripts/md_to_dialogues.py --md-root data/md --output data/raw/md_dialogues.json
python scripts/prepare_dataset.py --input data/raw/md_dialogues.json --output-dir data
```

Or from legacy curated JSON:

```powershell
python scripts/prepare_dataset.py --input data/raw/daisy_curated.json --output-dir data
```

Copy `train.jsonl` and `val.jsonl` into `training/` before an Azure ML job, or set `TRAIN_FILE` / `VAL_FILE` paths in the job environment.

## Books → Markdown (reference corpus)

PDF / FB2 / TXT / RTF / DOCX under `Eng/`, `Рус/`, etc. can be converted to **`data/md/`** (mirrored paths) for easier chunking and dialogue synthesis:

```powershell
pip install -r scripts/requirements-books.txt
python scripts/books_to_markdown.py --input-dir data --output-dir data/md
```

Legacy **`.doc`** is skipped unless converted to **`.docx`**. Output is one `.md` per source file with a `# title` heading and extracted plain text.
